# 검색 동작 문서

## 개요
이 프로젝트의 검색은 벡터 검색이 아닌 **규칙 기반 텍스트 검색**입니다.
핵심 구현 파일:
- `src/lib/notices.ts`
- `src/server/notices/notice-service.ts`

핵심 함수:
- `extractSearchTerms` (토큰화 + 불용어 제거)
- `filterNotices` (검색 통과 조건)
- `scoreNotice` (결과 랭킹)

## 파이프라인
1. 질의 정규화
- 공백 정리
- 소문자 변환

2. 토큰 추출 (`extractSearchTerms`)
- 정규식 `/[^\p{L}\p{N}]+/u`로 분리
- 불용어 제거 (`SEARCH_STOP_WORDS`)
- 숫자가 아닌 1글자 토큰 제거
- 순서 보존 중복 제거

3. 필터링 (`filterNotices`)
아래 중 하나라도 만족하면 통과:
- 질의 전체 문장 포함
- 공백 제거(compact) 문장 포함
- 의미 토큰 매칭 수가 기준 이상 (현재 `min(2, terms.length)`)

4. 점수화/정렬 (`scoreNotice`)
- 제목/요약/태그/본문 매칭 가중치 기반 점수
- 점수 동률이면 최신 날짜 우선

## 토큰화 상세

### 사용 정규식
`/[^\p{L}\p{N}]+/u`

의미:
- `\p{L}`: 유니코드 문자(Letter)
  - 한글/영문 포함
- `\p{N}`: 유니코드 숫자(Number)
- `[^\p{L}\p{N}]`: 문자/숫자가 아닌 구분자
  - 공백, 쉼표, 점, 괄호, 슬래시, 하이픈 등
- `+`: 구분자가 연속되어도 한 번에 분리
- `u`: 유니코드 속성 문법 활성화

결과적으로 **문자+숫자 덩어리만 토큰으로 남습니다.**

## 불용어 제거 상세

### `SEARCH_STOP_WORDS`
검색 의미가 약한 단어를 제거합니다.
예:
- 한국어: `정보`, `관련`, `문의`, `알려줘`, `요약`, `최신`
- 영어: `show`, `find`, `latest`, `info`

### 추가 제거 규칙
- 숫자가 아닌 1글자 토큰 제거
  - 예: `a`, `가`

### 안전장치
불용어 제거 후 토큰이 0개가 되면 원본 토큰(`rawTokens`)을 사용합니다.
- 검색어가 완전히 사라져 0건만 나오는 상황을 방지

## 예시

### 예시 1
입력:
`공모전 정보 알려줘`

1) 토큰화 결과:
`["공모전", "정보", "알려줘"]`

2) 불용어 제거 결과:
`["공모전"]`

### 예시 2
입력:
`airbus 101 공모전 정보 알려줘`

1) 토큰화 결과:
`["airbus", "101", "공모전", "정보", "알려줘"]`

2) 불용어 제거 결과:
`["airbus", "101", "공모전"]`

## 기존 방식 대비 개선점
이전에는 공백 분해 토큰을 전부 만족해야 하는 AND 조건(`every`)이 강해서,
`공모전 정보 알려줘` 같은 자연어 질문이 0건으로 떨어지기 쉬웠습니다.
현재는 불용어 제거 + 완화 매칭으로 누락을 줄였습니다.

## 튜닝 포인트
- 불용어 추가/삭제: `SEARCH_STOP_WORDS`
- 최소 토큰 매칭 기준: `requiredMatches`
- 점수 가중치: `scoreNotice`
